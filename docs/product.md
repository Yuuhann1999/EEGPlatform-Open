# Product

## Register

product

## Users

EEG researchers, students, and analysts who need to inspect uploaded EEG files, run common preprocessing operations, and generate ERP, PSD, topomap, and TFR views in a browser-based workflow. In the public deployment, users need a lightweight demonstration surface that is clear about resource limits and file requirements.

## Product Purpose

EEGAnalysis Pro helps users load EEG data, review signal quality, apply preprocessing steps, and inspect analysis views without managing scripts for every operation. Success means users can understand the current data state, run supported steps deliberately, and recover from session or backend limits without guessing what happened.

## Brand Personality

Calm, technical, reliable. The product should feel like an analysis workstation: dense enough for repeated use, restrained in styling, and explicit about processing state.

## Anti-references

Do not make it feel like a marketing landing page, decorative AI dashboard, or consumer wellness app. Avoid vague success/error copy, oversized visual effects, gratuitous animation, and UI that hides backend limits behind generic loading states.

## Design Principles

1. Show the current data state before asking users to act.
2. Make every long-running operation visibly accountable.
3. Prefer precise scientific labels over decorative language.
4. Keep controls dense but readable for repeated analytical work.
5. Explain public deployment limits in product language, not infrastructure jargon.

## Accessibility & Inclusion

Use semantic controls, visible focus states, and clear status messages. Do not rely on color alone for file, session, or processing state. Keep motion minimal and state-driven.
